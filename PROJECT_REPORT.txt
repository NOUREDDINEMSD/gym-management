1. WHAT IS THIS PROJECT?
================================================================================

You built a website to manage a gym. It has two sides:

  A) PUBLIC WEBSITE (anyone can visit)
     - Professional homepage with gym branding
     - Membership plans (Basic, Standard, Premium)
     - Dark theme with gold accents

  B) STAFF AREA (login required)
     - Dashboard with stats and charts
     - Mark member attendance (present / absent)
     - Search members by ID or name
     - Register, edit, and delete gym members

Think of it like:
  - The homepage = marketing for the gym
  - The staff area = daily work (attendance, members, reports)

3. HOW LARAVEL WORKS (SIMPLE EXPLANATION)
================================================================================

When someone visits a URL, Laravel follows this flow:

  1. ROUTE (routes/web.php)
     "When user visits /members, call MemberController@index"

  2. CONTROLLER (app/Http/Controllers/)
     Gets data from database, runs logic, passes data to a view.

  3. MODEL (app/Models/)
     Talks to database tables (Member, Attendance, etc.)

  4. VIEW (resources/views/)
     Blade file = HTML shown in the browser.

  5. MIGRATION (database/migrations/)
     Creates or changes database tables.

  6. SEEDER (database/seeders/)
     Inserts sample/fake data for testing.

Example: User visits /offers
  Route -> MembershipController@index -> Membership model -> memberships/index.blade.php

9. STEP 5 - MEMBERS SEARCH + REGISTER MEMBER
================================================================================

GOAL: Search members by ID/name. Staff can add new gym members.

IMPORTANT - TWO TYPES OF "REGISTRATION":
----------------------------------------

  STAFF REGISTRATION (Breeze):
    URL: /register
    Table: users
  Who: People who LOGIN to manage the gym (you, manager, front desk)

  GYM MEMBER REGISTRATION (your feature):
    URL: /members/create
    Table: members
  Who: People who WORK OUT at the gym (customers)

WHAT WAS DONE:
--------------

  1) MemberController:
       index()  -> search page (?q= for ID or name)
       create() -> registration form
       store()  -> save new member, auto ID (GYM007, GYM008...)

  2) Views:
       members/index.blade.php   -> search + member cards
       members/create.blade.php  -> register form
       components/member-card.blade.php -> shows photo, stats, etc.

  3) Photo upload stored in storage/app/public/members/
       Command once: php artisan storage:link

  4) Member card shows:
       Photo (or initials), ID, phone, membership, join date,
       present/absent counts, attendance rate %

ROUTES (login required):
  GET  /members         -> members.index (search)
  GET  /members/create  -> members.create (form)
  POST /members         -> members.store (save)

HOW TO REGISTER A NEW GYM MEMBER:
  1. Login as staff
  2. Members -> "+ Register Member" OR /members/create
  3. Fill name, phone, plan, join date, optional photo
  4. Save -> auto member code assigned
  5. Member appears in search and attendance

HOW TO TEST:

  - Register new member -> shows on attendance page

10. STEP 6 - DASHBOARD + EDIT / DELETE MEMBER
================================================================================

GOAL: Admin dashboard with stats/chart. Edit and delete members.

DASHBOARD (/dashboard):
-----------------------

  DashboardController calculates:

  | Stat card           | Meaning                                      |
  |---------------------|----------------------------------------------|
  | Total Members       | Count of all rows in members table           |
  | Today Attendance    | Present check-ins today                      |
  | Active Plans Used   | How many plan types are used by members      |
  | Monthly Income      | Sum of monthly price of every member's plan  |

  Chart: Last 7 days of PRESENT attendance (bar chart, CSS only)
  Sidebar: Members count per plan (Basic, Standard, Premium)

EDIT MEMBER:
------------
  URL: /members/{id}/edit  (example: /members/1/edit)
  - Change name, phone, plan, join date, photo
  - Member ID (GYM001) does NOT change
  - Can remove or replace photo
  Route: PUT /members/{id} -> members.update

DELETE MEMBER:
--------------
  - On edit page, red "Delete member" section
  - Browser confirm popup
  - Deletes member AND their attendance history (database cascade)
  Route: DELETE /members/{id} -> members.destroy

  Also: Edit button added on each member card in search results.

FILES:
  - app/Http/Controllers/DashboardController.php
  - resources/views/dashboard.blade.php
  - resources/views/members/edit.blade.php
  - MemberController: edit(), update(), destroy()

11. DATABASE TABLES EXPLAINED
================================================================================

  users (from Laravel + Breeze)
  -----------------------------
  Staff who log into the system.
  Columns: id, name, email, password, etc.

  memberships
  -----------
  Gym plans (products you sell).
  Example row: Standard, $49, 3 months, list of benefits.

  members
  -------
  Gym customers.
  Linked to memberships via membership_id.
  member_code = public ID like GYM001.

  attendances
  -----------
  Daily check-in records.
  status = 'present' or 'absent'
  One row per member per day.

RELATIONSHIPS:
  memberships (1) -----> (many) members
  members (1) ---------> (many) attendances
  users = separate (staff only, not linked to members table)
